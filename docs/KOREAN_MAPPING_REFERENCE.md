# 한글 매핑 참조 문서

## 변수명 한글 매핑 표준

### 원칙
1. **명확성**: 변수명만 봐도 무엇을 의미하는지 명확히 알 수 있어야 함
2. **일관성**: 프론트엔드와 백엔드에서 동일한 한글명 사용
3. **직관성**: 사용자가 쉽게 이해할 수 있는 자연스러운 한글 표현
4. **간결성**: 불필요한 설명은 생략, 핵심만 표현

---

## 1. 연속형 변수 (Continuous)

### 인구통계
| 변수명 | 한글명 | 설명 |
|--------|--------|------|
| `age` | 연령 | 나이 (세) |
| `age_scaled` | 연령 (정규화) | 정규화된 연령 |
| `Q6_income` | 소득액 (만원) | 월평균 개인소득 (만원) |
| `Q6_scaled` | 소득 (정규화) | 정규화된 소득 |
| `education_level` | 학력 | 학력 수준 |
| `education_level_scaled` | 학력 (정규화) | 정규화된 학력 |

### 소비 패턴
| 변수명 | 한글명 | 설명 |
|--------|--------|------|
| `Q8_count` | 전자제품 수 | 보유 전자제품 개수 |
| `Q8_count_scaled` | 전자제품 수 (정규화) | 정규화된 전자제품 수 |
| `Q8_premium_index` | 프리미엄폰 지수 | 프리미엄 제품 선호도 지수 |

### 라이프스타일
| 변수명 | 한글명 | 설명 |
|--------|--------|------|
| `drinking_types_count` | 음주 유형 수 | 경험한 음주 종류 개수 |
| `smoking_types_count` | 흡연 유형 수 | 경험한 흡연 종류 개수 |

## 5. 단위 매핑

### 금액
- `Q6_income`, `avg_income` → `만원`

### 나이
- `age`, `avg_age` → `세`

### 개수
- `Q8_count`, `avg_electronics_count` → `개`
- `drinking_types_count`, `smoking_types_count` → `개`
- `avg_children_count` → `명`

### 비율/지수
- `Q8_premium_index`, `avg_premium_index` → (단위 없음, 차트에서 % 표시)
- `college_graduate_rate`, `has_children_rate`, `metro_rate`, `premium_car_rate` → `%`

### 이진형 변수
- 모든 이진형 변수 → (단위 없음, 차트에서 %로 표시)

## 6. 구현 위치

### 프론트엔드
- `src/ui/profiling-ui-kit/components/comparison/utils.ts`
  - `FEATURE_NAME_KR`: 변수명 → 한글명
  - `CATEGORY_VALUE_KR`: 범주형 값 → 한글명
  - `FEATURE_UNIT`: 단위 매핑

### 백엔드
- `server/app/clustering/compare.py`
  - `FEATURE_NAME_MAP`: 변수명 → 한글명
  - `get_feature_display_name()`: 한글명 반환 함수
